# [인프라 11편] 카오스 엔지니어링 — 운영 환경에서 의도적으로 장애를 주입하여 복원력(Resilience)을 어떻게 검증할 것인가?

카오스엔지니어링, 복원력검증, 장애주입, SteadyState, ChaosMesh, 성능테스트, 인프라설계

&nbsp;

복잡한 분산 아키텍처에서 장애는 '발생하느냐'의 문제가 아니라 '언제 발생하느냐'의 문제입니다. 장애를 기다리는 대신 **미리 장애를 주입하여 약점을 찾아내는 과학적 실험**, 카오스 엔지니어링의 실전 전략을 분석합니다.

&nbsp;

---

&nbsp;

# 1. 실험 설계와 정상 상태(Steady State) 정의

&nbsp;

카오스 엔지니어링은 무작위 파괴가 아니라 정교한 실험입니다.
1. **지표 설정**: 시스템이 정상일 때의 지표(응답 시간 99th, 에러율 등)를 명확히 정의합니다.
2. **가설 수립**: "특정 DB 리전에 지연이 발생해도 사용자에게는 캐시된 데이터가 보여야 한다"는 가설을 세웁니다.

&nbsp;

&nbsp;

---

&nbsp;

# 2. 안전한 장애 주입과 가드레일

&nbsp;

운영 환경에서 사고를 내지 않기 위한 제어 장치가 필수입니다.
- **Blast Radius 제어**: 실험 대상을 전체가 아닌 특정 카나리 그룹이나 내부 테스트 계정 트래픽으로 한정합니다.
- **자동 중단 스위치**: 실험 도중 정의된 정상 상태 지표가 임계치를 넘어서면, 즉시 실험을 중단하고 롤백하는 자동화 시스템을 구축합니다.

&nbsp;

&nbsp;

---

&nbsp;

# 3. 실전 엔지니어링 교훈: 실패를 통한 성장

&nbsp;

카오스 엔지니어링의 가장 큰 가치는 **'알 수 없는 실패(Unknown-Unknowns)'**를 수면 위로 끌어올리는 것입니다. 장애 상황에서 관찰 도구가 제 기능을 하는지, 담당자가 적절히 대응하는지를 실습함으로써 조직 전체의 복원력을 상향 평준화할 수 있습니다. 시스템의 신뢰는 실험의 횟수에 비례합니다.

# [인프라 11편] GPU 클러스터 최적화 — 대규모 AI 추론 서비스의 리소스 부족과 비용 문제를 어떻게 해결할 것인가?

&nbsp;

생성형 AI의 열풍은 인프라 세계의 주인공을 CPU에서 GPU로 바꾸어 놓았다. 하지만 현실은 가혹하다. NVIDIA의 고성능 GPU는 구하기가 하늘의 별 따기이며, 클라우드 대여 비용은 시간당 수 달러를 훌쩍 넘긴다. 대규모 LLM(대형 언어 모델) 서비스를 운영하는 조직에게 GPU는 더 이상 단순한 하드웨어가 아니라, **'현명하게 소비해야 하는 비싼 소모품'**이 되었다. 

&nbsp;

본 글에서는 단 1%의 유휴 시간도 허용하지 않는 **고집적 GPU 클러스터링** 전략과 비용 효율적인 추론 인프라 설계 기법을 상세히 분석한다.

&nbsp;

&nbsp;

&nbsp;

# 1. 쪼개고 나눈다: Fractional GPU와 MIG의 마법

&nbsp;

하나의 LLM 모델이 GPU 하나를 통째로 차지하는 방식은 자원 낭비의 전형이다. 특히 실시간 요청이 적은 시간대에 비싼 A100/H100을 놀리는 것은 인프라 비용을 공중에 뿌리는 것과 같다.

&nbsp;

- **MIG (Multi-Instance GPU)**: NVIDIA의 MIG 기술을 활용하면 하나의 물리 GPU를 최대 7개의 독립된 하드웨어 인스턴스로 논리적으로 분할할 수 있다. 각 인스턴스는 독자적인 고립된 메모리와 연산 자원을 가지므로, 여러 개의 작은 모델을 하나의 GPU에서 간섭 없이 서빙할 수 있다.
- **Time-Slicing vs 하드웨어 분할**: 단순히 소프트웨어적으로 시간을 쪼개 쓰는 방식보다, 하드웨어 레벨에서 메모리를 격리하는 MIG가 추론 지연 시간의 안정성(Predictable Latency) 측면에서 훨씬 유리하다.

&nbsp;

&nbsp;

# 2. 스케줄링의 혁신: Karpenter와 Kueue의 만남

&nbsp;

쿠버네티스의 표준 오토스케일러는 초단위로 급증하는 AI 워크로드를 감당하기에 너무 느리다. 

&nbsp;

- **Karpenter를 통한 즉각적 프로비저닝**: 작업 대기열을 실시간으로 감시하고, 가장 적합한 GPU 노드를 60초 이내에 띄워 워크로드를 배치한다. 작업이 끝나면 즉시 노드를 반납하여 비용 발생 구간을 최소화한다.
- **Bin Packing 전략**: 여러 GPU 워크로드를 최대한 적은 수의 노드에 밀집시켜 노드 점유 수를 줄인다. 이때 모델 가중치의 로딩 시간을 줄이기 위해 **가중치 프리페칭(Pre-fetching)**과 **로컬 캐싱** 전략이 병행되어야 한다.

# 3. 실전 엔지니어링 교훈: VRAM은 현금이다

&nbsp;

AI 인프라를 설계하는 시니어 엔지니어는 모델의 파라미터 크기뿐만 아니라 **KV Cache**의 증가량까지 고려하여 VRAM 사용량을 정밀하게 계산해야 한다.

1. **스팟 인스턴스(Spot Instance)의 적극 활용**: 추론 워크로드는 학습보다 내결함성이 높다. 모델 체크포인트와 로드밸런서를 연동하여 스팟 인스턴스 회수 시 즉시 트래픽을 타 노드로 돌리는 설계를 통해 비용을 70% 이상 절감하라.
2. **양자화(Quantization)와 추론 엔진 최적화**: vLLM이나 Triton 같은 엔진을 도입하여 처리량(Throughput)을 높이는 것은 인프라를 늘리는 것보다 훨씬 경제적이다.

AI 시대의 인프라 경쟁력은 "누가 더 많은 GPU를 가졌는가"가 아니라, **"누가 GPU 1개당 더 많은 요청을 처리하는가"**에서 결정된다.
